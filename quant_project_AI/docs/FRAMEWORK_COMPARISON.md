## 概念词典（初学者必读）

> 本报告涉及大量量化交易专业概念。请先阅读本章，遇到不理解的术语可随时回来查阅。

### A. 基础概念

**回测 (Backtest)**
用历史数据模拟交易策略的表现。比如你有一个"当5日均线上穿20日均线就买入"的想法，回测就是用过去几年的真实价格数据，假装你从第一天开始按这个规则交易，看最终能赚多少钱。回测的目的不是证明策略能赚钱，而是在投入真金白银之前发现问题。

**策略 (Strategy)**
一套完整的交易规则，告诉你"什么时候买、什么时候卖、买卖多少"。比如"RSI低于30就买入，高于70就卖出"就是一个简单策略。本框架包含18种不同类型的策略。

**做多 (Long) / 做空 (Short)**
做多 = 先买后卖，赌价格上涨。你100元买入，120元卖出，赚20元。
做空 = 先卖后买，赌价格下跌。你先"借"股票以100元卖出，等价格跌到80元再买回来还给别人，赚20元。做空的风险理论上是无限的（价格可以无限上涨）。

**杠杆 (Leverage)**
用借来的钱放大你的交易。比如你有1万元，用10倍杠杆就可以控制10万元的仓位。价格涨1%，你赚10%（1万元变1.1万）；但价格跌1%，你也亏10%（1万元变9千）。杠杆放大收益的同时也放大风险。

**止损 (Stop-Loss)**
预设的"最大可承受亏损"。比如设置-20%止损意味着：当这笔交易亏损达到20%时，系统自动卖出以防止更大亏损。就像给你的亏损装一个"刹车"。

**OHLCV 数据**
每根K线包含的5个数字：O=开盘价 (Open)，H=最高价 (High)，L=最低价 (Low)，C=收盘价 (Close)，V=成交量 (Volume)。这是量化交易最基础的数据格式。

### B. 性能指标（衡量策略好不好）

**年化收益 (Annualized Return)**
把策略的总收益换算成"每年能赚多少"。如果一个策略3年赚了60%，年化收益大约是17%。这样不同时间长度的策略就可以公平比较了。

**最大回撤 / Max Drawdown (DD)**
策略从最高点跌到最低点的最大跌幅。比如你的账户从100万涨到150万，又跌到90万，最大回撤 = (150-90)/150 = 40%。它衡量的是"最坏的时候你要亏多少"。对投资者来说，这个数字比收益更重要——你能承受40%的账户缩水吗？

**Sharpe Ratio（夏普比率）**
衡量"每承担一份风险能获得多少收益"。公式简化理解：Sharpe = 收益 / 波动性。
- Sharpe < 1：每承担1份风险，获得不到1份收益，不太好
- Sharpe 1-2：可接受
- Sharpe 2-3：很好
- Sharpe > 3：优秀（但要警惕是否过拟合）

**Calmar Ratio（卡尔玛比率）**
年化收益 / 最大回撤。比如年化20%、最大回撤10%，Calmar = 2。它比Sharpe更直观地回答一个问题："为了赚到这个收益，我最多要忍受多大的亏损？"

**OOS (Out-of-Sample) 收益**
"样本外"收益。策略参数是用一段历史数据（训练集）找到的，OOS收益是用另一段策略"从未见过"的数据测试得到的。这比用训练数据得到的收益更可信，因为它模拟了真实的"未来"表现。

**WFE (Walk-Forward Efficiency)**
Walk-Forward效率 = OOS收益 / 训练集收益。如果训练集赚50%但OOS只赚5%，WFE = 10%，说明策略严重过拟合（只在训练数据上表现好）。WFE越接近100%，策略越可能在未来也有效。

### C. 防过拟合概念（防止"自欺欺人"）

**过拟合 (Overfitting)**
策略在历史数据上表现极好，但在新数据上表现很差。就像考试前把答案全背下来，考试100分，但换一套题就0分。这是量化交易最大的陷阱——你以为找到了赚钱秘诀，实际上只是把噪音当成了规律。

**Walk-Forward 优化 (WF)**
把历史数据分成多个时间段，每次用前面的数据训练参数，用后面的数据测试。比如用2020年数据找参数，用2021年数据测试；再用2021年数据找参数，用2022年数据测试。这模拟了真实的"在过去训练，在未来交易"的过程。本框架使用6个这样的窗口。

**Embargo（隔离期）**
训练数据和测试数据之间插入的"空白期"（本框架为5根K线）。为什么需要？因为相邻的数据往往有相关性，如果训练数据的最后一天和测试数据的第一天挨着，训练阶段可能已经"偷看"到了一些测试数据的信息。Embargo就是在中间挖掉几天，确保训练和测试完全隔离。

**Validation Gate（验证门控）**
在训练集和测试集之间加一道"安检"。如果策略在训练集上赚了+50%，但在验证集上亏了-30%，说明参数严重过拟合。此时系统会自动将这组参数的测试结果归零，不让它混入最终排名。这是本框架独有的机制。

**Deflated Sharpe Ratio (DSR)**
普通Sharpe的问题：你测试了1000种参数，总会有几个看起来Sharpe很高的——但那可能只是运气。DSR通过统计学方法校正"测试了太多参数"带来的假阳性。DSR p-value > 0.5 表示这个策略的Sharpe大概率不是碰巧得到的。

**Monte Carlo 模拟**
给历史价格加上微小的随机噪音（比如每天价格波动±0.1%），生成30条"可能的历史"。如果策略在这30条路径上都赚钱，说明它不是依赖于某个特定的价格走势，而是真的有逻辑。

**Block Bootstrap（分块自助法）**
把历史数据切成20天一块，然后随机重新排列组合，生成20条新的价格序列来测试策略。它比Monte Carlo更激进——直接打乱了时间顺序，检验策略是否只依赖于特定的事件顺序。

**OHLC Shuffle（K线随机重排）**
把每天的开高低收四个价格随机重新分配。如果策略在这种"混乱"的数据上也能赚钱，那可能策略根本没有真正的预测能力（因为连价格结构都打乱了还能赚，说明是偶然的）。

**Bonferroni 校正**
你同时测试了100种策略，有5种看起来赚钱。但根据概率论，即使所有策略都无效，也会有5%看起来赚钱（纯运气）。Bonferroni校正就是把"赚钱的标准"提高100倍（因为你测了100种），确保找到的真的是有效的。

**Look-ahead Bias（前瞻偏差）**
在回测中"偷看"了未来的信息。最常见的错误：用今天的收盘价来决定今天的买卖——但真实交易中，收盘价在收盘后才知道。本框架使用 Next-Open（用下一根K线的开盘价成交）来严格避免这个问题。

**Survivorship Bias（幸存者偏差）**
只用"活下来"的股票做回测。比如你测试2010-2020年的策略，但只用2020年还在交易的股票，那些中途退市、破产的股票被排除了。这会让回测结果偏乐观，因为你"事后"只选了赢家。

### D. 交易成本概念（为什么成本建模很重要）

**佣金 (Commission)**
每次买卖交易所收取的手续费。Binance收0.04%（买10000元收4元），IBKR收0.05%。看起来很少，但如果策略每天交易，一年下来成本可能吃掉大部分利润。

**滑点 (Slippage)**
你下单要买100元的股票，实际成交价可能是100.03元（因为你的买单推高了价格）。这个0.03元的差额就是滑点。杠杆越高，需要的实际资金规模越大，对市场的冲击也越大，所以本框架的滑点随杠杆动态增加。

**市场冲击 (Market Impact / Almgren-Chriss)**
大单对价格的影响。你要买100万元的股票，不可能一笔买完——你买的过程中价格已经被你自己推高了。Almgren-Chriss模型是学术界广泛使用的市场冲击模型，能更真实地模拟大单的实际成交价。大多数回测框架完全忽略了这个因素。

**资金费 / Funding Rate**
加杠杆做多相当于"借钱买"，你需要每天支付利息。在加密货币期货市场，这叫资金费率，通常每8小时收一次。本框架模拟了这个成本，并且考虑到高杠杆的费率更高（因为借款风险更大）。50倍杠杆的日资金费高达3%，一年1,095%——这意味着你的策略必须每天赚超过3%才能覆盖成本。

**止损滑点 (SL Slippage)**
当价格快速下跌触发止损时，实际卖出价可能比止损价更低。市场恐慌时流动性枯竭，你设了-20%止损，实际可能亏-22%。本框架模拟了这种"快市额外损失"。

**固定手数 vs 全仓复利**
大多数回测框架假设你赚了钱就全部投入下一笔交易（复利）。这在回测中会产生惊人的回报（利滚利），但现实中交易者通常固定每笔交易的金额。本框架的 `_deploy()` 函数模拟了固定手数交易——你不会因为前面赚了钱就把全部利润都投入冒险。

### E. 技术架构概念

**Numba / @njit**
一种让Python代码运行速度接近C语言的技术。正常Python非常慢（每秒几百次回测），但通过@njit编译后，同样的代码可以达到每秒36,000次。这对于需要测试2760万种参数组合的大规模扫描至关重要。

**Next-Open 执行**
回测中的成交方式：今天收盘后产生信号，明天开盘价成交。这是最贴近现实的方式——因为你不可能在看到收盘价的同时以收盘价交易。很多框架默认用"当前收盘价成交"，这是一种前瞻偏差。

**TCA (Transaction Cost Analysis)**
交易成本分析，把每笔交易的成本拆解为：佣金多少、滑点多少、市场冲击多少。帮你理解"钱都花在哪了"，是专业机构的标配工具。

**RAG (Retrieval-Augmented Generation)**
一种让AI能够检索和使用你自己的数据来回答问题的技术。在量化交易中，RAG可以让策略参考新闻、研报等非结构化信息，同时通过 as_of_date 机制确保回测时不会"偷看"未来的新闻。

**Parquet / Arrow / Mmap**
三种高效的数据存储格式。Parquet是列式存储，读取特定列非常快；Arrow是内存中的列式格式，支持零拷贝；Mmap是内存映射文件，让程序像读内存一样读硬盘。这些技术让数据加载速度比传统CSV快10-100倍。

**Protocol-Based / SOLID**
软件设计原则。SOLID确保代码的每个模块只做一件事（单一职责），模块之间通过"接口"而非"继承"连接。好处是可以轻松替换任何组件——比如把回测的"模拟成交器"换成真实交易所的"真实成交器"，其他代码不用改。

## 四、架构优势总结

### 4.1 本框架独有能力 (其他框架不具备)

| # | 能力 | 技术细节 | 价值 |

## 六、结论

本框架在**防过拟合体系**和**成本建模真实性**两个维度上达到了接近顶级量化机构的水平，远超所有主流开源框架。具体而言：

1. **防过拟合**: 10层检验体系在开源领域独一无二，40.7% 的策略通过 Deflated Sharpe 显著性检验
2. **成本真实性**: 6种动态成本因素 (佣金/滑点/冲击/资金费/SL滑点/仓位约束) 使回测结果不失真
3. **回测速度**: 36,000/s 的吞吐量使大规模参数搜索成为可能 (2760万次回测仅需768秒)
4. **统计严谨**: 每个策略都有 Sharpe、DSR p-value、年化收益等机构级指标

主要差距在**交易频率** (日级 vs tick级) 和**实盘基础设施** (需开发交易所连接器)，这是下一阶段的优化方向。
