## 十、技术架构分析

### 性能关键技术

| 技术 | 效果 | 说明 |
|:-----|:-----|:-----|
| Numba `@njit(cache=True, fastmath=True)` | 编译为原生机器码 | 消除 Python 解释器开销，首次编译后缓存 |
| 预计算指标数组 | 避免重复计算 | MA/EMA/RSI/ATR/KAMA 一次计算，所有参数组合共享 |
| Lemire O(N) 滚动极值 | 算法级优化 | rolling max/min 从 O(N×W) → O(N) |
| 信号-填充-PnL 融合循环 | 消除中间分配 | 单个 for 循环完成全部逻辑，无 NumPy 临时数组 |
| Grid 缓存 | 消除重复转换 | `list[tuple]` → `np.ndarray` 转换只做一次 |
| `prange` 并行扫描 | 多核利用 | 参数网格内部 Numba 自动并行 |
| ThreadPoolExecutor | 策略级并行 | Numba 释放 GIL，策略间可并行 |
| 连续内存布局 | 缓存友好 | `np.ascontiguousarray` 确保 L1/L2 缓存命中 |

### 反过拟合技术栈 (11层)

| 层 | 技术 | 作用 |
|---:|:-----|:-----|
| 1 | Purged Walk-Forward | 6窗口时间序列交叉验证，训练/验证/测试严格分离 |
| 2 | Multi-Metric Scoring | 综合考虑收益、回撤、交易次数的复合评分 |
| 3 | Minimum Trade Filter | 最低交易次数门槛 (≥20)，排除低频噪声 |
| 4 | Validation Gate | 验证期表现不能偏离训练期太远 |
| 5 | Cross-Window Consistency | 参数须在多个时间窗口都有效 |
| 6 | Monte Carlo Price Perturbation | 对 OOS 数据加随机噪声重复测试 |
| 7 | OHLC Shuffle | 打乱 OHLC 角色测试策略对价格结构的依赖 |
| 8 | Block Bootstrap | 块状重采样保留序列依赖性 |
| 9 | Deflated Sharpe Ratio | 校正多重假设检验偏差 |
| 10 | Composite Ranking | 综合所有验证层的最终排名 |
| 11 | CPCV | 组合清洗交叉验证，最大化数据利用率 |

## 十一、总结

### 核心性能指标

- **单回测延迟**: 25μs (39,870 backtests/sec)
- **全策略扫描**: 8,440 组合 → 69ms
- **扫描吞吐**: 122,359 combos/sec
- **WF 优化 (18策略)**: 272ms

### 相对其他框架

- 比 Backtrader 快 **1994x**
- 比 vectorbt 快 **20x**
- 比 QuantConnect LEAN 快 **199x**

### 独特优势

1. **全 Numba 原生编译**: 18个策略全部编译为机器码，无 Python 回退
2. **11层反过拟合**: 业界最全面的参数验证体系，远超同类框架
3. **信号一致性**: 实盘信号直接调用同一 Numba kernel，100% 与回测一致
4. **亚毫秒级延迟**: 单次回测 < 100μs，适合实时策略评估和高频参数搜索
5. **真实成本建模**: 手续费 + 滑点 + 资金费率 + 借贷成本 + 杠杆 + 止损，8维成本模型
